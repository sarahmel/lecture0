Git clone

La première chose à faire est de créer un clone du fichier sur lequel on va travailler. C'est un repository. Dans l'invite de commande, on va dans le fichier où il y aura notre fichier git. (Rapelle cd Users => permet d'aller dans le dossier Users, puis cd sarah, puis cd Onedrive\Bureau\Git\Mon premier projet Git\)
Puis on tape "git clone" suivi du lien qu'on a dans notre compte Git.

Git add

La commande git add, permet de prendre en compte des changements dans les files que l'on va indiquer, à chaque fois qu'on veut faire un "commit", c'est à dire, à chaque fois que l'on veut sauvegarder une version sur Git.

Par exemple, si je veux sauver les différentes versions, à chaque fois que je le modifie, de cette page html, je vais écrire dans ma ligne de commande: git add hello.html.

Maintenant, à chaque fois que je vais faire des modifications, Git va enregistrer les versions. Cela peut marcher pour des fichiers html, mais également.py si je bosse sur python, etc...

Le ou les fichiers modifiés (et déjà indexés) grâce à la commande add, sont alors "prêts à être commités", c'est-à-dire prêts à être inclus dans le prochain commit.

Git commit

Du point de vue du développeur qui utilise "git", l'évolution d'un projet est ainsi composé d'une succession de "commits", commentés, qui correspondent à autant de modifications des fichiers du projet, commentées par leur auteur.

Par comparaison avec l'état précédent du projet, un commit peut être considéré comme la matérialisation d'une série de modifications que l'on souhaite enregistrer dans le projet : d'un point de vue très "pratique", un commit peut être vu comme une étape de plus dans l'avancement d'un projet.

Un commit peut aussi bien consister en la modification d'un seul caractère, qu'en l'ajout d'une librairie complète au projet., cela ne dépend que de la manière de les utiliser !

Git status

Plutôt facile: en entrant la commande git status, on obtient le statut de notre git.

Git push

Maintenant, il faut envoyer ces changement sur Github pour les partager. On le fait grâce à la commande "git push". On nous demande d'entrer l'identifiant (sarahmel) et le mdp (@ch...8)

Git pull

Si je veux downloader une version sur laquelle quelqu'un d'uatre a travaillé, à partir de github, je vais utiliser la commande git pull

Merge conflicts

Parfois, quand on fait des changements simultanés avec des gens sur internet, il peut y avoir des merge conflicts. On a un message d'erreur qui apparaît: Merge conflict in "dossier" You need to fix the conflict and commit (pas mot à mot mais en gros). Si on va ensuite voir sur le github, ce qui est au dessus de HEAD, c'est mon changement. Il suffit d'enlever les lignes que l'on veut pas (la ligne HEAD, la ligne avec le nom du conflit + tout ce qu'on ne veut pas dans le code. On a résolu le merge conflict.

Git log

Donne la liste des commits, des auteurs et du temps du commit. C'est l'historique des commits.

Git reset

Pour revenir à un ancien commit

Exemple:

  • git add hello.html : on a ajouté le fichier au git
  • git commit -m "Fix marge conflicts": là j'ai fait le commit que j'ai nommé "fix..." pour pouvoir le retrouver si besoin
  • git push : permet de pousser les changements vers le github, sinon les changements sont uniquement au niveau local.
  • git log : permet de voir l'historique de tous les commits que j'ai faits.
  • git reflog: permet de voir l'historique des commits avec leur nom
