Créer un clone

La première hose à faire est de créer un clone du fichier sur lequel on va travailler. C'est un repository. Dans l'invite de commande, on va dans le fichier où il y aura notre fichier git. (Rapelle cd Users => permet d'aller dans le dossier Users, puis cd sarah, puis cd Onedrive\Bureau\Git\Mon premier projet Git\)
Puis on tape "git clone" suivi du lien qu'on a dans notre compte Git.

Choisir les fichiers que l'on veut tracker

La commande git add, permet de prendre en compte des changements dans les files que l'on va indiquer, à chaque fois qu'on veut faire un "commit", c'est à dire, à chaque fois que l'on veut sauvegarder une version sur Git.

Par exemple, si je veux sauver les différentes versions, à chaque fois que je le modifie, de cette page html, je vais écrire dans ma ligne de commande: git add hello.html.

Maintenant, à chaque fois que je vais faire des modifications, Git va enregistrer les versions. Cela peut marcher pour des fichiers html, mais également.py si je bosse sur python, etc...
